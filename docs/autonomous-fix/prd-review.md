## The central cross-cutting finding (the synthesizer's headline)

**THE ENFORCEABILITY PARTITION (R6-A, reinforced by R3-5A/6C, R2-C1).** This is the single finding that, unaddressed, hollows the skill. A markdown discipline cannot *hard-fail* runtime behavior — it can only advise; only the wired template/runtime can refuse. The draft implied all nine ACs hard-fail. The truth:

- **Mechanical (statically enforceable by the vendored conformance-lint, on the template YAML/prompt):** AC2 (fence shape incl. title-inside-markers), AC5 (cost-cap *present*), AC6 (no auto-merge / draft-only), AC7a (default-deny allowlist present), AC4 (gate-names present in the prompt — a grep). These *fail in CI*.
- **Advisory-at-the-discipline-layer / enforced only by the wired runtime + branch-protection primitives:** AC1 (sensitive-path refuse), AC3 (reproduce-first), AC7b (read-only-first ordering), AC8 (no leak in commit/comment), AC9-behavioral-parity. The discipline *tells* the agent; the *runtime* (path-check step that aborts, branch-protection that blocks merge, secret-scan that fails the commit) *enforces*. AC6 is the strongest because GH branch-protection is a primitive *outside* the agent; AC1 is the weakest because its check sits *downstream of the agent it guards* (no independent oracle — the dual-LLM pattern §9 cites is the fix the PRD consciously declines, now stated as an accepted limit).

**[FIXED]** — the revised PRD adds a **"Enforcement surface per AC"** table partitioning all ten ACs mechanical-vs-runtime-vs-advisory, states the AC1 no-independent-oracle limit honestly (with the dual-LLM trade-off accepted), and §6 + §3 are reconciled to it. This converts the skill from a *safety suggestion* back into a *safety contract* where the contract is real (template-lint + branch-protection primitives) and the advisory parts are *labeled* as such rather than over-promised.
