## PASS 1 — Acceptance criteria observable · **CONDITIONAL→FIXED**

All 7 ACs name a fail-condition (the bar). Three precision defects let vagueness through; all fixed.

- **B1 (Review 1) — AC1's no-secrets criterion is semantic but its test (S1.2) is a syntactic regex proxy.** A 32-char Datadog key has no `key=` prefix and isn't "long" relative to a 33-char repro command; the grep both misses real secrets and false-positives on benign config. AC1 inherits the "autonomous-fix AC8 no-leak" citation but **not** its detector (the gate's AC8 leak-scan is explicitly project-supplied — `autonomous-fix/prd.md` AC8). **FIX APPLIED:** AC1 restated as a **value-shape allowlist** — every field's value must match its declared kind's expected shape (`kind`∈enum, `cli`∈known-command, `service`∈identifier, `secret_ref`∈`[A-Z][A-Z0-9_]*` env-var NAME, `mcp`∈`mcp__*` prefix); any value failing its field's shape fails AC1. This makes "no credential value" decidable rather than aspirational. S1.2 reworded to test the allowlist, not a regex.
- **B2 (Review 1) — AC2's "an axis that has both available" is an unobservable precondition; S1.3 quietly tests only field-presence.** "Both available" is ground truth about the harness, not the artifact. The no-silent-drop failure direction had **no scenario**. **FIX APPLIED:** AC2 decomposed into (a) **field-presence** (always emit both fields, empty allowed — S1.3) and (b) **no-silent-drop** (given a fixture environment manifest declaring what's available, every available binding is recorded — new **S2.1**, failure direction). AC2 now scoped to the **routed axes only** (see B-scope below).
- **m4 (Review 6) — AC4 asserts "one contract" while U2 says the contract count is undecided.** AC4 then says "a tracker exposes fetch_issue+comment, an obs source exposes query_logs(window)" — that's two interfaces. **FIX APPLIED:** AC4 reworded to "each axis's adapters conform to **that axis's** capability contract" — no longer asserting a resolution U2 holds open.
